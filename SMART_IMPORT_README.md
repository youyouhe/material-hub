## ⚡ 5分钟快速测试

### 步骤1: 配置环境（1分钟）

```bash
# 确保有 .env 文件并配置了 LLM
cat .env | grep DEEPSEEK_API_KEY

# 如果没有，复制模板并编辑
cp .env.example .env
vim .env  # 设置 DEEPSEEK_API_KEY=sk-your-key
```

### 步骤2: 启动服务（1分钟）

```bash
# 使用启动脚本
./start.sh

# 或手动启动
cd backend && python main.py &
cd frontend && npm run dev &
```

### 步骤3: 访问系统（30秒）

```
浏览器打开: http://localhost:5173
登录: admin / admin123
```

### 步骤4: 测试导入（2分钟）

1. 点击导航栏 **"智能导入"** 按钮
2. 准备一个测试文件（营业执照、证书照片等）
3. 拖拽文件到上传区
4. 点击 **"开始智能导入"**
5. 观察处理结果

**预期**:
- ✅ 文件名清晰的应显示 "自动归档"
- ⚠️ 文件名模糊的应显示 "待审核"

### 步骤5: 审核测试（1分钟）

如果有待审核项：
1. 点击 **"审核队列"** 按钮
2. 选择待审核项
3. 查看识别结果
4. 点击 **"确认归档"**

## 🐛 常见问题

### Q: 上传后一直显示"处理中"

**A**: 检查后端日志
```bash
tail -f backend.log | grep ERROR
```

### Q: 提示"LLM分析失败"

**A**: 检查API配置
```bash
# 验证环境变量
echo $DEEPSEEK_API_KEY

# 测试连接
cd backend
python -c "from llm_provider import get_llm_provider; llm = get_llm_provider(); print(llm.chat('test'))"
```

### Q: 文件预览显示404

**A**: 检查临时文件
```bash
ls -lh data/temp/
# 应该能看到上传的文件
```


### 测试重点

1. **文件名清晰** → 应自动归档
2. **文件名模糊** → 应进入审核
3. **内容清晰** → 识别准确
4. **内容模糊** → 需人工确认
